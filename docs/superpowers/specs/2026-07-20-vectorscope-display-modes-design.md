# Vectorscope Display Modes — Design

**Date:** 2026-07-20
**Status:** Approved

## Summary

Add three independently persisted display modes to every Vectorscope panel and Dock
Vectorscope:

```txt
Lissajous
Polar Sample
Polar Level
```

`Lissajous` is the existing X/Y phase trace, renamed from the informal product term
“Trace.” `Polar Sample` folds the same real L/R samples into an upper semicircle and renders
them as a short-persistence point cloud. `Polar Level` aggregates the same samples by polar
angle and renders a filled stereo-energy envelope.

The first slice is frontend-only. It reuses the existing request-keyed vectorscope history
slab (~40 ms rows, ~25 updates/s) for both Polar modes and leaves the Rust audio/DSP/IPC path
unchanged. The existing high-rate Rust-generated SVG path remains the Lissajous live source.

## Product Decisions

- Each Vectorscope panel stores its own mode and restores it after restart.
- Multiple Vectorscope panels may show different modes simultaneously.
- Mode selection lives in the existing panel settings menu.
- Settings order is `Mode`, `Channel pair`, then mode-specific controls.
- Mode labels have no explanatory subtitle.
- Existing and newly created panels default to `Lissajous`.
- Mode changes are immediate; there is no crossfade or shape morph.
- Correlation remains visible in all three modes.
- Polar views do not show ±45° safety guides in this slice.
- Polar views show the selected channel labels at the left and right ends.
- All modes use stable automatic magnification.
- Polar Sample uses a fixed ~400 ms afterglow.
- Polar Level uses a filled envelope with an outline and fast-attack/slow-release motion.
- Polar Level exposes a per-instance `Peak hold` toggle, default off.
- Peak hold is indefinite and resets through the existing global Clear action.
- Silence fades Polar Sample to empty, collapses Polar Level, clears correlation through the
  existing signal gate, and freezes automatic magnification so noise is not expanded.
- Snapshot mode preserves the selected display mode.
- Dock Vectorscope supports and persists all three modes independently.
- The existing hold-to-slow gesture remains Lissajous-only.
- Lissajous `Dots` and `Heatmap` styles are out of scope for this slice.
- All modes reuse the current theme-driven Vectorscope color.

## Terminology and Stored Controls

Add the following full-panel controls:

```txt
vectorscopeMode: "lissajous" | "polarSample" | "polarLevel"
vectorscopePolarLevelPeakHold: boolean
```

Defaults:

```txt
vectorscopeMode = "lissajous"
vectorscopePolarLevelPeakHold = false
```

Both controls belong to `panelControlsById`, follow panel-instance persistence, preset restore,
duplication, and normalization, and do not affect other Vectorscope instances.

The Dock `correlation` module receives equivalent controls:

```txt
mode: "lissajous" | "polarSample" | "polarLevel"
polarLevelPeakHold: boolean
```

The existing module id remains `correlation`; renaming that module is unrelated to this feature.

## Analysis Identity

Display mode and Peak hold are frontend display choices. They must not enter the vectorscope
analysis request key.

```txt
vectorscope:pair:<x>:<y>
```

Two panels showing the same pair in different modes share one Rust `VectorscopeMeter`, one live
result, and one history slab. The existing maximum of four unique vectorscope requests therefore
continues to apply to channel pairs, not display modes.

## Existing Data Sources

### Lissajous live

Keep the existing `vectorscopeResultsByKey[key].path` generated by Rust at the live frame cadence.
This preserves current behavior and keeps this slice away from the audio/DSP pipeline.

### Polar live

Read the existing `VectorscopeHistorySlab` through
`getVectorscopeHistoryForKey(requestKey)`. Each row contains real interleaved pairs:

```txt
[L0, R0, L1, R1, ...]
```

Rows arrive at approximately 40 ms cadence. The current frame/context render cadence is allowed
to be higher; Polar rendering only changes when the slab version changes.

### Snapshot

Extend `resolveVectorscopeSnapshotForKey` to return the selected row’s `pairs` as well as the
existing derived Lissajous `path`, correlation, and energy metrics. A snapshot never fabricates
missing samples.

Snapshot behavior:

- Lissajous renders the selected row as today.
- Polar Sample renders the selected row as a point cloud without temporal fading.
- Polar Level aggregates the selected row and renders its settled envelope immediately.
- The runtime Peak hold outline is hidden in snapshot mode. Peak hold represents the live period
  since Clear and must not be presented as historical state that was never stored.

## Shared Polar Projection

For each real sample pair, clamp L and R to `[-1, 1]`, then rotate to Mid/Side coordinates:

```txt
mid  = (L + R) / sqrt(2)
side = (R - L) / sqrt(2)
```

Fold signal polarity into the upper semicircle:

```txt
polarX = side
polarY = abs(mid)
radius = hypot(polarX, polarY)
angle  = atan2(polarX, polarY)  // -π/2 ... +π/2
```

Interpretation:

- centered mono maps to the top center regardless of waveform polarity;
- left/right imbalance moves toward the corresponding side;
- opposite-polarity content approaches the semicircle ends;
- radius retains the sample magnitude before display magnification.

No ±45° guides or danger-region fill are drawn.

## Stable Automatic Magnification

Automatic magnification is shared conceptually by all three modes but held independently by each
panel instance.

For Polar modes, derive the target extent from the maximum projected radius in the active sample
window. Maintain a display extent with time-based behavior:

- when target extent grows, accept it immediately so the drawing shrinks before clipping;
- when target extent falls, release toward it slowly so the drawing expands without pumping;
- clamp to the existing vectorscope extent floor and preserve plot padding;
- when the selected pair has no signal, keep the last display extent instead of magnifying noise;
- initialize from the first valid signal window.

The implementation must be time-based rather than frame-count-based so 25 Hz and irregular UI
frames behave consistently. Exact release tuning is an implementation constant covered by pure
tests and visual verification; it is not a user setting in this slice.

Lissajous keeps its existing Rust-side extent hold. A later payload unification may move all three
modes to the same frontend extent implementation, but that is not required here.

## Polar Sample

Polar Sample uses Canvas 2D and redraws from a bounded history window.

- Select rows no older than 400 ms relative to the newest slab timestamp, not wall-clock time.
- Draw every selected real pair as a point; do not connect points and do not interpolate them.
- Newest points use the current Vectorscope trace token at high opacity.
- Opacity falls continuously with row age and reaches zero at the window edge.
- Point size scales with device pixel ratio and is clamped for small Dock plots.
- All rows in one draw use one shared automatic magnification extent.
- On silence, stop adding visible points; the existing rows fade out naturally.
- Snapshot mode draws its single selected row at full point opacity.

The fixed 400 ms persistence is intentionally not exposed as a setting in this slice.

## Polar Level

Polar Level aggregates the same projected points into a fixed number of angular bins across the
semicircle. Use a constant bin count sufficient for a smooth envelope (initial target: 64 bins).

For each window:

1. assign each projected sample to an angular bin;
2. accumulate squared radius in that bin;
3. normalize by the total sample count, then take the square root so bin length represents its
   contribution to window RMS energy;
4. apply a small fixed neighboring-bin smoothing kernel to avoid a jagged polygon without
   inventing temporal samples;
5. apply the shared automatic magnification;
6. update the displayed bin envelope with fast attack and slow release.

Attack/release behavior is time-based:

- bins grow quickly when new energy exceeds the displayed value;
- bins shrink more slowly when energy falls;
- snapshot mode bypasses temporal smoothing and displays the selected row’s aggregate directly;
- silence drives the envelope toward zero while leaving the magnification extent frozen.

Render one closed filled Canvas path plus its outline. Fill and outline use the existing
Vectorscope trace token with different alpha; no new mode colors are introduced.

## Polar Level Peak Hold

When `vectorscopePolarLevelPeakHold` is enabled:

- keep a per-bin maximum of the displayed Polar Level envelope;
- render the maxima as an unfilled thin outline;
- maxima do not decay;
- toggling Peak hold off hides and discards the held values;
- toggling it on starts a fresh hold from the current envelope;
- changing channel pair or mode discards held values for that panel;
- the existing global Clear action discards held values in every full-panel and Dock instance;
- Peak hold values are runtime-only and are not persisted or placed in history;
- snapshot mode hides the runtime hold outline.

The boolean toggle is persisted; the measured held values are not.

## Clear Integration

Expose a monotonically increasing frontend reset epoch through the shared meter data context. It
increments only after the existing Clear action successfully clears the active source.

Polar renderers observe this epoch and reset:

- Polar Level attack/release state;
- Polar Level Peak hold bins;
- Polar Sample local draw state if any;
- automatic magnification initialization where appropriate.

This is a display reset signal only. Existing history and backend Clear behavior remains
authoritative.

## Full-Panel UI

The Vectorscope settings group is:

```txt
Mode          Lissajous | Polar Sample | Polar Level
Channel pair  <existing options>
Peak hold     Off | On  // Polar Level only
```

The mode selector uses labels only. There are no descriptions. The plot switches immediately.

Plot behavior:

- Correlation rail and readout retain their current position and behavior in every mode.
- Lissajous retains the current diagonal grid and corner pair labels.
- Polar modes replace the diagonal grid with a subtle semicircle baseline/arc only.
- Polar modes show the selected first-channel label at the left end and second-channel label at
  the right end; no `Center` label is shown.
- Lissajous alone owns the existing hold-to-slow pointer gesture.

## Dock UI

Dock Vectorscope supports the same modes and mode-specific Peak hold toggle through
`DockModuleSettings`.

- The Dock mode and Peak hold state are independent of all full-panel instances.
- Standard and expanded Dock layouts retain the existing correlation rail/readout.
- Polar plots use the same math and renderer as full panels.
- At very small plot sizes, hide endpoint labels before reducing the plot itself.
- Mode switching is immediate.

## Rendering Structure

Keep data math separate from Canvas operations:

- pure projection/window/bin/extent/envelope functions live in a math module and are the primary
  test surface;
- a reusable Polar plot module owns Canvas sizing, DPR handling, theme-token resolution, and
  drawing;
- full-panel and Dock adapters supply controls, pair labels, live/snapshot rows, signal state,
  and reset epoch;
- the existing Lissajous SVG implementation remains in its current adapters for this slice.

Canvas is selected for Polar modes because point persistence and a repeatedly updated filled
envelope are bounded pixel rendering tasks. SVG path strings remain only for existing Lissajous.

## Performance Bounds

- Polar Sample reads at most the fixed 400 ms window (~10 rows, ~2,000 points with current
  history sampling).
- Polar Level uses the same bounded window and a fixed-size bin array.
- Do not copy slab pair arrays; consume `rowAt()` subarray views.
- Recompute geometry only when the slab version, snapshot row, plot size, controls, or reset epoch
  changes.
- Do not add allocations, locks, or syscalls to the audio callback.
- Do not add a Rust/IPC payload in this slice.

## Testing

Pure math tests cover:

- folded Mid/Side projection for centered, left, right, and opposite-polarity pairs;
- 400 ms row selection relative to newest timestamp;
- stable extent immediate shrink / slow expansion / silence freeze;
- Polar Sample age-to-alpha mapping;
- Polar Level bin assignment, RMS-energy normalization, neighboring-bin smoothing;
- time-based fast attack and slow release;
- Peak hold accumulation and reset.

Control and persistence tests cover:

- defaults and invalid-value normalization;
- per-instance workspace persistence and preset restore;
- display mode and Peak hold excluded from analysis request keys;
- Dock defaults, normalization, serialization, and reset-to-default behavior.

UI tests cover:

- settings order and mode-specific Peak hold visibility;
- immediate mode switching;
- correct renderer and grid per mode;
- endpoint labels in Polar modes;
- Correlation present in all modes;
- snapshot mode preserving the selected mode;
- Peak hold hidden in snapshot mode and reset by Clear;
- hold-to-slow active only in Lissajous;
- Dock standard/expanded mode rendering and small-size label hiding.

## Out of Scope

- Lissajous Dots or Heatmap styles.
- User-adjustable persistence, attack, release, point size, bin count, or auto-scale speed.
- ±45° Polar safety guides or danger coloring.
- Mode-specific colors.
- Mode-transition animation.
- Rust/IPC changes or a 60 Hz Polar payload.
- Frequency-dependent Stereo Cloud or width-history meters.
- Persisting runtime Peak hold values.

## Acceptance Criteria

- Existing users see unchanged Lissajous output after upgrade.
- Every full-panel and Dock Vectorscope independently stores and restores its mode.
- Polar Sample is visibly continuous at the existing history cadence and fades to empty on silence.
- Polar Level is stable, reacts quickly to new energy, releases smoothly, and does not visibly
  jitter under steady input.
- Automatic magnification does not visibly breathe under steady input, clips neither sudden loud
  input nor expands silence/noise.
- Peak hold is optional, defaults off, holds indefinitely, and resets through Clear.
- Snapshot and Dock behavior matches this spec.
- No Rust audio/DSP/IPC code changes are required.
